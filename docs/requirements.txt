13. RECOMMENDED DOCKER IMAGES
- Backend build/runtime image:

14. RECOMMENDED CONTAINER ARCHITECTURE
- 1 container for backend NestJS API
- 1 container for frontend build/runtime
- 1 container for PostgreSQL
- 1 persistent Docker volume for PostgreSQL data
- Optional reverse proxy container for TLS and routing

15. DEPLOYMENT NOTES
- The repository root contains a generic Dockerfile and compose files, but the full solution is a monorepo with backend and frontend workspaces.
- For production on AlmaLinux, a multi-stage Docker strategy is recommended so backend, frontend, and PostgreSQL can be deployed separately.
- Prisma requires DATABASE_URL to be available at build and migration time where applicable.
- PostgreSQL persistence must be mapped to a Docker volume or host path.
- Prisma Studio and Node inspector ports should not be exposed publicly in production.
